Linux Hardware Fix · Kubuntu / Ubuntu

Getac S410 G4 — Physical Trackpad Buttons Not Working

Tested on Kubuntu 24.04 (Wayland). Applies to any Ubuntu-based distro where the ELAN0700 touchpad is misdetected as a clickpad.

Symptom

After installing Kubuntu (or another Ubuntu-based distro) on the Getac S410 G4, the physical left and right buttons below the trackpad do not register clicks in the OS, even though the trackpad surface itself works fine for moving the cursor.

Root Cause

Running libinput list-devices reveals the real problem:

event5  - ELAN0700:00 093A:0274 Touchpad:
  kernel bug: missing right button, assuming it is a clickpad.

libinput is misidentifying the trackpad as a clickpad (a buttonless pad you physically depress). This causes it to ignore events from the separate physical buttons entirely. The fix is to add a libinput quirks file that tells it the correct hardware type.

1

Verify the issue with libinput

Confirm libinput is reporting the clickpad misdetection:

libinput list-devices

You should see the kernel bug: missing right button, assuming it is a clickpad message next to the ELAN0700 touchpad entry. Also confirm button events do reach the kernel (even though the OS ignores them):

sudo apt install evtest
sudo evtest

Select the ELAN0700 device and press the physical buttons — you should see events logged, confirming it is a driver config issue, not a hardware failure.

2

Create the libinput quirks directory

The /etc/libinput/ directory does not exist by default. Create it:

sudo mkdir -p /etc/libinput
3

Create the quirks override file

sudo nano /etc/libinput/local-overrides.quirks

Paste in the following content exactly:

/etc/libinput/local-overrides.quirks 
[Getac S410 Touchpad]
MatchName=ELAN0700:00 093A:0274 Touchpad
ModelTouchpadButtonAreas=1
Saving in nano: Press Ctrl+O → Enter to confirm filename → Ctrl+X to exit.
4

Reboot

sudo reboot

After rebooting, the physical left and right trackpad buttons should work normally.

5

Verify the fix

Run libinput list-devices again and confirm the "clickpad" warning is gone for the ELAN0700 device.

Why this works

The ModelTouchpadButtonAreas=1 quirk tells libinput to treat the bottom edge of the trackpad as discrete left/right button zones, matching the physical hardware layout of the S410 G4. Without this, libinput's clickpad assumption swallows the button events before they reach the desktop.

Automated Script

A bash script and desktop launcher are included in this project to apply the fix without manually opening a terminal.

A

Option 1 — Double-click launcher (no terminal needed)

In your file manager, navigate to the project folder and double-click:

Getac-Touchpad-Fix.desktop

If prompted, choose Run. A terminal window will open automatically, run the fix, and wait for you to press Enter before closing.

If it won't run: Right-click the file → Properties → tick Allow executing file as program, then try again.
B

Option 2 — Run from terminal

If you prefer to run it manually from a terminal:

bash /A_Dev/Getac_Touchpad_Fix_Linux/fix_touchpad.sh
C

What the script does

The script automates all manual steps above:

1. Checks libinput is installed (installs it if missing)
2. Detects whether the clickpad misdetection is present
3. Creates /etc/libinput/ and writes local-overrides.quirks
   (backs up any existing file first)
4. Prompts you to reboot — or skip and reboot manually later
